1. INTRODUCTION
1.1. Context of the study
Over the last decade, we have seen significant changes in the field of web development, driven
by advances in web frameworks, increased computer performance and bandwidth, and shifting user
expectations. One of them started in 2015, when two Google employees, developer Alex Russell and
designer Frances Berriman, introduced the concept of Progressive Web Applications (PWAs). They
saw PWAs as a way to bring the best user experiences from native mobile apps to the web by using
features such as service workers and web app manifests.
The usage and popularity of the new web application standard have increased significantly since it
was introduced. Flipkart, India’s largest e-commerce platform, was one of the first to use this technology
in their application, Flipkart Lite. This new application dramatically improved the user experience
compared to the previous mobile platform. The adoption of the new standard resulted in users spending
three times more time on the website, a 40% higher reengagement rate, a 70% higher conversion
rate among users who reached the site using the "Add to Homescreen" function, and a three times
lower data usage.
Other major companies have recognized the value of PWAs and have joined the trend. Platforms
such as Twitter, Forbes, Alibaba, and many others benefited significantly, with lower data consumption,
significant increases in conversions, monthly active users, as well as faster loading times and higher
user sessions and engagement.
The advantages of PWAs are not limited to large corporations, as they can offer significant benefits
to startups by accelerating product development and reducing costs. Unlike traditional native apps,
they can be developed, tested, and deployed quickly, allowing startups to iterate through ideas cost-
effectively. This makes them a powerful tool for companies that want to accelerate the development
of prototypes and effectively match product to market. Using PWAs, startups can allocate resources
more efficiently to innovation and refinement of their offerings, rather than investing heavily in native
app development.
Given the numerous benefits provided by PWA, the technology has been widely adopted since its
introduction in 2015. However, when deciding to adopt this web standard, developers face a crucial
question: which framework and programming language to use? Traditionally, JavaScript has been
the main language for web development, and frameworks such as Angular, React, and Vue.js were
designed to facilitate PWA creation. However, the web development environment is changing and new
technologies are emerging that can provide attractive alternatives.
This thesis examines Blazor technology as one of the alternatives to JavaScript frameworks in PWA
development. Blazor, a technology developed by Microsoft, is a modern front-end web framework that
allows developers to create interactive web applications using mainly C# and .NET, while still allowing
for integration with JavaScript. It supports two hosting models: server-side and WebAssembly. The
server-side model performs application logic on the server and handles the user interface via a real-time 
SignalR connection, whereas Blazor WebAssembly runs code client-side in the browser on a
WebAssembly-based .NET runtime.
